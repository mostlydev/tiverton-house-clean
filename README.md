# Tiverton House

This repo is the source of truth for a live AI-managed trading desk. It runs a small roster of specialized agents — traders, a coordinator, a news router, and a fleet monitor — operating through a shared system of record and a set of agreed-upon rules.

The desk is built on [Clawdapus](https://clawdapus.com), an agent orchestration platform. If you're new to Clawdapus, the concepts here will make more sense after reading the Clawdapus docs. If you're already familiar, this repo is a full working example of how a production pod is structured.

## The Agents

| Agent | Role |
|-------|------|
| `tiverton` | Desk coordinator — compliance, traffic, escalation |
| `weston` | Momentum trader |
| `logan` | Value trader |
| `gerrard` | Macro and longer-horizon trader |
| `danforth` | Credit and special situations trader |
| `dundas` | News triage and routing |
| `sentinel` | Fleet health and desk monitoring |

Each agent has a contract under `agents/<name>/` that defines its identity, focus, and rules. Shared behavior (startup sequence, tool conventions, escalation) lives in `agents/_shared/`.

## How It's Organized

The repo has three distinct layers, and it's important to keep them straight.

**Source layer** — what you edit and commit:
- `claw-pod.yml` — the full pod topology: agents, feeds, services, env wiring
- `agents/` — agent contracts and identities
- `scripts/` — wrapper scripts agents and operators use at runtime
- `policy/` — desk rules, risk limits, approval boundaries
- `services/trading-api/` — the Rails system of record for all desk state
- `docs/` — operator and conceptual documentation

**Compiled runtime layer** — generated by `claw up -d`, never hand-edited:
- `compose.generated.yml`
- `.claw-runtime/`

**Persistent local state** — real runtime data that stays off git:
- `.claw-auth/`, `.claw-memory/`, `.claw-session-history/`, `.claw-state/`
- Live `storage/` contents and `var/`

The rule: if it should survive a `claw up` rebuild, it goes in source. If it's live desk state, it lives in `trading-api`. If it's an agent's working notes, it lives in its private surface.

## The System of Record

`trading-api` (under `services/trading-api/`) owns everything that matters for trading:

- wallets and positions
- trade proposals, approvals, fills
- watchlists
- feed payloads served to agents

Agents don't maintain their own shadow copies of this state. They read and write through the API (via wrapper scripts) and receive fresh market context through injected feeds on every inference call. Agent memory is for notes and reasoning — not for tracking positions.

## How Context Gets Into Agents

Market data, desk risk, and channel context reach agents through feed injection, not through markdown files or plugins. Before each LLM call, `cllama` (the model proxy) fetches the feeds that agent is subscribed to, wraps each one in a labeled block, and appends them to the system prompt. The agent sees fresh data every 30–300 seconds depending on the feed.

Feed subscriptions are declared per-agent in `claw-pod.yml`. The feeds themselves are served by:
- `trading-api` — market context, momentum signals, value context, desk risk
- `claw-wall` — recent `#trading-floor` Discord messages
- `claw-api` — fleet health and alerts

## First 10 Minutes on a Live Machine

```bash
cd <repo-root>
./scripts/verify-pod.sh
claw compose ps
claw compose logs tiverton --tail 50
claw compose exec -T trading-api curl -fsS http://127.0.0.1:4000/up
```

If the pod isn't running yet:

```bash
./scripts/bootstrap/init-pod-env.sh --force
./scripts/verify-pod.sh
./scripts/pod-up.sh
```

## Common Tasks

### Recompile and restart the pod

```bash
./scripts/pod-up.sh
```

Run this after changing `claw-pod.yml`, any OpenClawfile, agent contracts, or feed wiring.

### Stop the pod

```bash
./scripts/pod-down.sh
```

### Check service health

```bash
claw compose ps
claw compose logs <service> --tail 50
claw compose exec -T trading-api curl -fsS http://127.0.0.1:4000/up
```

### Read live desk state

```bash
claw compose exec -T trading-api curl -fsS http://127.0.0.1:4000/api/v1/market_context/logan
claw compose exec -T trading-api curl -fsS http://127.0.0.1:4000/api/v1/desk_risk_context/tiverton
```

### Watch agent logs

```bash
claw compose logs weston --tail 50
claw compose logs cllama --tail 100
```

## Committing Changes

Safe to commit:
- `claw-pod.yml`, `agents/`, `scripts/`, `policy/`, `docs/`
- `services/trading-api/` (app code, specs, config, migrations)
- Build files: `OpenClawfile`, `Dockerfile`, `Dockerfile.sidekiq`

Never commit:
- `.env` or any secret file
- `.claw-auth/`, `.claw-memory/`, `.claw-session-history/`, `.claw-state/`, `.claw-backups/`
- `.claw-runtime*/`, `compose.generated.yml`
- `agent-memories/`, live `storage/` contents, `var/`
- Generated descriptors like `services/trading-api/.claw-describe.json`
